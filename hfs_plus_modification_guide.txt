File creation

Volume Header:
  - lastMountedVersion <--- modify this (with a unique constant) so we know the utility did the modifications
  - modifyDate
  - fileCount
  - freeBlocks
  - nextAllocationBlock
  - nextCatalogId
  - writeCount <--- important so changes aren't rolled back
  
Journal:
  - end <--- where we start writing our transactions, needs to be updated afterwards
  - start ?--- modify if we overflowed back to the beginning of the buffer, advancing previous oldest transaction
